What I'd build next (priority order)
1. create_bug MCP tool + CLI command
When publish-test-results finds failures, let agents auto-file ADO bugs:


ado-sync publish-test-results --create-bugs-on-failure
MCP: create_bug({ title, failureDetails, testCaseId, buildId })

This unblocks Use Case 4 — the GitHub Agentic Workflow that says "on failures, Playwright Healer Agent proposes a fix PR". The bug creation is the handoff between the CI result and the healer.

2. CTRF format support in publish-test-results
The scope doc and materia article both use CTRF. It's becoming the standard cross-framework report format. Teams using the materia pattern will arrive with CTRF output expecting ado-sync to accept it.

3. get_story_context MCP tool (Planner agent feed)
The current get_work_items tool returns raw ADO fields. The Planner agent in Use Case 3 needs a purpose-built output:


{
  "storyId": 1234,
  "title": "...",
  "acceptanceCriteria": "...",      // HTML stripped, bullet-formatted
  "suggestedTags": ["@smoke"],
  "suggestedPermissions": [],       // parsed from AC text
  "relatedTestCases": [456, 789]    // existing TCs already linked
}
This is what feeds the Planner → Generator chain at 80% first-run pass rate.

4. ai-workflow-manifest.json generator
A new command:


ado-sync generate --manifest --story-ids 1234
Produces a .ai-workflow-manifest.json in the e2e/bdd/ folder using the materia pattern — required documents checklist, validation steps, output paths. This gives any team the structured AI memory that materia built manually.

5. GitHub Actions reusable workflow
The scope doc explicitly asks for this:

"There is need to write GitHub Actions templates in TR Org Workflows repo"

A reusable workflow file:


# .github/workflows/ado-sync.yml (reusable)
on:
  workflow_call:
    inputs:
      test-result-path: ...
      create-bugs-on-failure: ...
Teams subscribe to it — one-line adoption in their own pipelines.

6. Flaky test detection in publish-test-results
Materia's 4-method flaky detection:

Summary-level flaky count
Retry pattern analysis
Explicit marking
Passed-after-retry
Add a --detect-flaky flag that outputs a flaky test report alongside the ADO publish. Tag flaky TCs in ADO with ado-sync:flaky so you can track trends across runs.
